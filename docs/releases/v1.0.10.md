# LSJ-AutoGLM v1.0.10

## 变更摘要

- 在系统提示词中明确 0–999 归一化坐标协议、正反换算公式，并加入 1080×2400 的具体换算示例，避免模型混用截图像素与归一化坐标。
- 将动作的 `ActionSpec.description` 以及参数类型、必填性、说明和示例完整渲染进系统提示词。
- 模型屏幕上下文新增截图宽高与坐标空间声明。
- 新增机器可解析的 `[ACTION_TRACE]` 日志，记录动作前后的应用/页面状态、模型坐标、转换后的物理坐标、截图尺寸和执行结果。
- 新增 ADB 前台页面状态采集，并标注页面标题来源；对无法读取可见标题的自绘界面使用前台 Activity 作为低延迟回退。
- 修复动作解析器误把 `Type` 文本内部 Markdown 代码围栏当作外层响应围栏的问题。
- 修复 Windows 上外部配置原子写入在时间戳和文件大小均未变化时可能无法被 GUI 检测的问题。

## 验证

- 自动化测试：44 项通过。
- 1080×2400 真机坐标校准通过：模型坐标 `[100, 240]` 可稳定转换为物理坐标 `(108, 576)`。
- 使用当前 MiMo 模型完成同屏多轮对照，确认归一化提示能阻止原始像素坐标泄漏；视觉目标定位误差仍会通过新增动作追踪日志暴露。
- Windows x64 隐藏启动冒烟测试、任务运行器、内置 adb/scrcpy、首次 `.env` 生成和敏感文件泄漏检查均通过。
- 目录原始体积：157.96 MiB；ZIP 体积：67.51 MiB；文件数量：272。
- ZIP SHA-256：`d0c1be6d124943c8099dec1c9dd59cb5dadcebbc473d0b13729e8b0cfaf7fb9a`。

## 分发说明

1. 下载 `OpenAutoGLM-GUI-v1.0.10-windows-x64.zip`。
2. 使用同名 `.sha256` 文件校验完整性后，完整解压 ZIP。
3. 运行目录中的 `OpenAutoGLM-GUI.exe`。
4. 首次启动会自动生成 `.env`，API Key 需由用户在设置页填写。

不要只复制 exe；`_internal` 目录是必需的运行时内容。
