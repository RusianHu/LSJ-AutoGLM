# ADB 连接与配对生命周期

设备页将“配对授权”和“当前传输连接”作为两层独立状态处理。

## 操作语义

- `pair`：Android 11+ 无线调试的首次信任建立。必须使用手机“使用配对码配对设备”窗口中的专用配对端口和当前 6 位码，或扫描本程序生成的二维码。
- `connect`：建立当前 ADB transport。传统 TCP/IP 和 Android 11+ TLS 的连接端口都可能显示为 `IP:端口`，不能仅凭外形判断协议。
- `disconnect`：只断开当前无线 transport，并清除本程序保存的自动重连地址；不会撤销手机中的配对授权。
- 解除配对：只能在手机“无线调试 -> 已配对的设备”中选择本电脑并执行“忘记”。
- `adb usb`：只用于由 `adb tcpip` 建立的传统无线 transport，不适用于 Android 11+ TLS 无线调试。

配对端口、TLS 连接端口和传统 `adb tcpip` 端口是三个不同概念。Android 11+ 的前两种端口可能在无线调试重启后变化。

## 传输兼容矩阵

| 传输 | 常见设备标识 | 配对 | 主动断开 | USB 一键转无线 | 恢复 USB 模式 |
|------|--------------|------|----------|----------------|---------------|
| USB 真机 | USB serial，如 `77eaf689` | 不需要 | 不支持，拔出数据线 | 支持 | 不适用 |
| 传统 TCP/IP | `IP:端口`，通常为 5555 | 不需要 | 支持 | 不适用 | 支持 |
| Android 11+ TLS mDNS | `adb-..._adb-tls-connect._tcp` | 二维码或配对码 | 支持，不解除配对 | 不适用 | 不支持 |
| Android 11+ TLS 显式端点 | `IP:动态端口`，同时存在对应 mDNS TLS 服务 | 二维码或配对码 | 支持，不解除配对 | 不适用 | 不支持 |
| 模拟器 | `emulator-端口` | 不需要 | 由模拟器管理 | 不支持 | 不支持 |

显式 `IP:端口` 若与 `_adb-tls-connect._tcp` 服务端点一致，会被二次识别为 TLS，避免错误启用“恢复 USB 模式”。若 ADB 同时列出同一 TLS transport 的 mDNS 名称和显式地址，设备页会合并重复项。

## 二维码配对流程

1. 生成独立的 `studio-*` 服务名和随机密码，二维码载荷为 `WIFI:T:ADB;S:<服务名>;P:<密码>;;`。
2. 检查 ADB mDNS 后，只匹配本次二维码的同名 `_adb-tls-pairing._tcp` 实例，不以“出现任意新设备”推断成功。
3. 对发现的专用配对端口执行 `adb pair`。
4. 先等待 ADB 自行建立 TLS transport，避免同时 `adb connect` 产生重复设备；等待失败时才连接同一主机发现到的 TLS 连接端点。

## 配对码流程

1. 用户填写手机配对码窗口当前显示的 `IP:专用配对端口` 和 6 位码。
2. 若同一主机仅发现一个 mDNS 配对服务，程序会自动纠正误填的连接端口。
3. `protocol fault` 通常表示端口不是专用配对端口、配对码过期或窗口已关闭，界面会返回可操作的错误说明。

## 并发与退出

- 配对、连接、断开、USB/TCP/IP 切换均在单一后台操作线程串行执行，设备轮询在操作期间暂停。
- 关闭对话框或应用时会中止等待，并终止仍在运行的 ADB 子进程。
- 用户主动断开 TLS transport 时会一并清除历史显式 IP 自动重连配置，防止轮询把设备立即连回。
- ADB 可执行文件选择顺序为环境变量覆盖、系统 PATH、项目内置版本；选中的 ADB 目录始终置于运行时 PATH 首位，避免客户端和 server 版本混用。
